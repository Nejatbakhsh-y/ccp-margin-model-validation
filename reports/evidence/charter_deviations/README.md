# Charter Deviation Evidence

This directory contains controlled records for deviations from the Independent
Validation Charter.

## Required Files

- `charter_deviation_register.csv`: Central register of all identified charter
  deviations.
- `DEV-YYYY-NNN.md`: Completed request and assessment for each deviation.
- `DEV-YYYY-NNN/`: Supporting evidence directory, when required.

## Deviation Identifier

Each deviation must use:

`DEV-YYYY-NNN`

Example:

`DEV-2026-001`

The completed deviation record should be named:

`DEV-2026-001.md`

Supporting evidence should be stored under:

`DEV-2026-001/`

## Required Evidence

Evidence may include:

- Deviation request.
- Charter requirement affected.
- Business justification.
- Risk assessment.
- Independent-validator assessment.
- Compensating-control design.
- Monitoring results.
- Model-use restrictions.
- Validation-authority decision.
- Related validation finding.
- Related change or revalidation assessment.
- Extension decision.
- Closure evidence.

## Independence

The model owner may request a deviation but cannot approve the deviation or
determine its effect on independent validation.

The independent validator assesses the effect on validation independence,
validation scope, findings, approval, and revalidation.

## Expiration

An expired deviation does not automatically close.

An expired and unresolved deviation must be treated as unapproved
noncompliance and evaluated for:

- A formal validation finding.
- Model-use restrictions.
- Partial or full revalidation.
- Suspension of production use.

## Evidence Preservation

Original requests, assessments, decisions, rejected requests, prior versions,
extensions, monitoring results, and closure records must remain
version-controlled.

Prior evidence must not be overwritten or deleted.
