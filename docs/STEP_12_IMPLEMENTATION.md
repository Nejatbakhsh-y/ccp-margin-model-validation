## Formula

```text
Total Initial Margin
= Base Margin
+ Liquidity Add-on
+ Concentration Add-on
+ Gap-Risk Add-on
+ Stress Buffer
```

The stress buffer is implemented as a residual coverage bridge:

```text
Stress Buffer
= max(
    Required Stress Coverage Ratio × Maximum Approved Stress Loss
    − Pre-Stress Margin,
    0
  )
```

This avoids using an arbitrary percentage uplift merely to improve backtesting.

## Installation procedure

1. Extract the bundle.
2. Copy the bundle's `src`, `tests`, and configuration snippet into the existing project root. Do not create a second nested project directory.

```yaml
  base_margin:
    var_col: your_actual_column_name
```

### Stress losses

```text
member_id
scenario_id
stress_loss
```

Each member must have at least one approved stress scenario.

## Governance controls implemented

- No silent fallback for unknown liquidity buckets.
- No silent fallback for unknown asset classes.
- All rates, thresholds, shocks, and stress coverage assumptions require a documented parameter source.
- Each component returns member-level and detailed attribution.
- Floors and caps are explicit and testable.
- Attribution is reconciled to each member-level component.
- Total margin is reconciled to the exact sum of components.
- Sensitivity tests verify expected monotonic behavior.
- Deterministic sorting and tie-breaking are used.
- Component limitations are embedded in module metadata.
